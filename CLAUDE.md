# SPLSHY widgets — brief

This repo hosts the runtime widget `.js` files served from splshy.com. Each is
a self-contained, zero-dependency IIFE (vanilla ES5-style JS — `var`, no build
step) that a host page loads via a `<script>` tag. The embed codes that load
them are generated by the **SPLSHY Studio** in the separate `getsplashy` repo
(`../getsplashy`, the builder + backend at getsplashy.com). This repo is pure
front-end runtime; all auth, storage, and APIs live in getsplashy.

## The four widgets
- **`infinite/widget.js`** (`sif-`) — the flagship reel carousel. LIVE on
  VisitRaleigh. Renders multiple desktop layouts via `desktopStyle` (see
  below). Variable number of reels.
- **`grid/widget.js`** (`sgr-`) — a grid of up to 12 video cards.
- **`single/widget.js`** (`srv-`) — single (or twin) reel player. LIVE.
- **`stories/widget.js`** (`sst-`) — Instagram-style highlight circles.

Don't mix the class prefixes up — each widget namespaces all its CSS/DOM with
its own prefix so multiple widgets can coexist on one host page.

## Infinite layout modes
Mobile vs desktop is gated by
`matchMedia("(min-width:768px) and (any-pointer:fine)")`. A `forceMobile`
config (and `.sif-force-mobile` class) forces the phone layout — that's what
the Studio's phone preview uses. On desktop, `cfg.desktopStyle` selects:
- **`carousel`** — horizontal reel row with prev/next arrows. Supports a
  **double-row "row-4" variant** (`--sif-visible:4`, `.sif-widget--row4`) that
  left-anchors and shows two rows of reels.
- **`accordion-3`** — centre card + 2 side cards (V=3 band).
- **`accordion-5`** — centre + 2 each side (V=5 band).

Accordion mechanics: cards are positioned by `rel = slot - centreSlot`; side
cards translate `cardW*0.68` / scale `0.69`, off-stage cards slide fully past
the `overflow:hidden` edge (pure translate, opacity stays 1 to avoid an edge
"flash"). Slide is `.52s cubic-bezier(.4,0,.2,1)` with opacity delayed `.26s`
so entry/exit hides behind the centre card. Carries via `ACCORDION_TRANS` /
`applyAccordionCard` / `isAccordionDesktop`. Desktop CSS pins
`.sif-card{transform:scale(1)!important;filter:brightness(1)!important}`, which
is why the accordion animates with inline `!important` transforms.

`cfg.startReel` opens the carousel centred on a chosen reel; the centred reel
is exposed on the DOM as `data-sif-current` for host pages to read.

## Config surface (`cfg.*`, read from the embed)
**Shared by all/most widgets:** `clientId`, `containerId`, `reels` (the video
array), `savedWidgetId`, `analytics` (set `false` to opt out), `analyticsEndpoint`,
`captionsEndpoint`, `allowedOrigins`, `igUrl`, `logoUrl`, `logoRing`,
`followerCount`.
**Centered heading + subheader** (optional, renders above the widget when
either is set): `centerHeading` / `centerSubheading` — on infinite, grid, and
stories.
**Infinite-specific:** `desktopStyle`, `startReel`, `forceMobile`, `autoAdvance`,
`hoverPreview`.
**Grid-specific:** `compact`, `shrinkDesktop`, `hideBottomRowDesktop`,
`autoplayIndices`, `hlsJsSrc`.
**Single-specific:** `videoUrl`, `posterUrl`, `label`, `captions`,
`captionsAvailable`, `hoverPreview`.
**Stories-specific:** `ringColor`.

## Analytics (two independent pipelines)
1. **Splshy-native** → POSTs batched events to
   `https://www.getsplashy.com/api/play/events` (override via
   `cfg.analyticsEndpoint`) using `sendBeacon` with a `fetch` keepalive
   fallback. Event types: `play` (fires once a reel hits ~3s), `watch`
   (seconds watched), `impression` (via IntersectionObserver). Requires
   `cfg.clientId`; harmless no-op without it. Opt out per-embed with
   `cfg.analytics === false`. Powers the getsplashy analytics dashboard.
2. **Host-page GA4** → `gaTrack()` fires `gtag("event", ...)` (GA4 direct) or
   `dataLayer.push()` (Google Tag Manager) to whatever the HOST page runs —
   independent of Splshy's pipeline, no-ops if neither is present. Events fire
   on deliberate user actions (open/play/progress/complete/error), not on
   always-on autoplay lanes, to avoid flooding GA. Both pipelines wrap every
   call so analytics can never break the widget.

## Captions
Multi-language VTT captions are fetched from `cfg.captionsEndpoint` (served by
getsplashy). The translation itself (es/fr/de/zh) happens server-side in the
getsplashy repo at video-save time; widgets just render the chosen track.

## Security
- **`safeUrl()`** in every widget sanitizes video/poster/link URLs (XSS
  hardening) before they hit the DOM — reels data is user-controlled.
- **`vercel.json`** sets response headers: `X-Content-Type-Options: nosniff`,
  `Referrer-Policy: strict-origin-when-cross-origin`, and a CSP
  `frame-ancestors 'self' https://www.getsplashy.com https://getsplashy.com`
  (so the widgets/preview can only be framed by the Studio).

## Deploy
`git add <files> && git commit -m "..." && git push`. Vercel auto-deploys;
if it doesn't, an empty commit (`git commit --allow-empty`) forces it.
There is a `test/` harness (`test/index.html`, `test/mobile`, `test/probe.html`)
for exercising the widgets without the host site.
